## はじめに

複数のプロジェクトを同時に開発していると、あちこちのターミナルを行き来して状況を確認するのが面倒になります。
「全プロジェクトのターミナルを一画面で俯瞰できたら便利なのに」と思ったことはないでしょうか。

今回紹介する **`ws`（Workspace Manager）** は、tmux を使って複数の作業セッションを作成し、それらを別モニターにタイル表示でミラーリングできるシェルスクリプトです。

## ws コマンドでできること

- **各プロジェクトのターミナルを tmux セッションとして管理** できる
- **別モニター（別ターミナル）にダッシュボードとして一覧表示** できる
- IDE のターミナルで操作した内容がダッシュボード側に **リアルタイムで反映** される
- セッションの追加・削除時にダッシュボードが **自動更新** される

## 前提条件

- tmux がインストールされていること
- bash が利用可能なこと

```bash
# tmux のインストール（Ubuntu/Debian の場合）
sudo apt install tmux

# macOS の場合
brew install tmux
```

## セットアップ

以下のスクリプトを `ws` という名前で PATH の通ったディレクトリに保存し、実行権限を付与します。

```bash
# 例: ~/bin に配置
curl -o ~/bin/ws https://raw.githubusercontent.com/nisioka/myTools/main/shell/ws
chmod +x ~/bin/ws
```

## 使い方

### サブコマンド一覧

| コマンド | 説明 |
|---------|------|
| `ws add [name]` | tmux セッションを作成して attach（名前省略時はカレントディレクトリ名） |
| `ws rm [name]` | セッションを終了（名前省略時はカレントディレクトリ名） |
| `ws list` | ワークスペースの一覧表示 |
| `ws dash` | ダッシュボードを開く |
| `ws help` | ヘルプを表示 |

### 基本的な使い方

各プロジェクトの IDE ターミナルで `ws add` を実行し、別モニターで `ws dash` を開きます。

```bash
# プロジェクトA のターミナルで
cd ~/projects/api
ws add
# → セッション "ws-api" が作られて attach される

# プロジェクトB のターミナルで
cd ~/projects/web
ws add
# → セッション "ws-web" が作られて attach される

# 別モニター / 別ターミナルで
ws dash
# → 全セッションがタイル表示でミラーされる
```

### セッション名を指定する

カレントディレクトリ名以外の名前を付けたい場合は、引数で指定できます。

```bash
ws add my-project
```

### ワークスペースの一覧を確認

```bash
ws list
# ワークスペース:
#   api  (/home/user/projects/api)
#   web  (/home/user/projects/web)
```

### セッションを終了する

```bash
ws rm api
# ✓ 'api' を終了しました
#   ダッシュボードを更新しました
```

## ダッシュボードの操作

ダッシュボードは tmux のセッションとして動作しているため、通常の tmux 操作が使えます。

| 操作 | 説明 |
|------|------|
| `Ctrl+B` → 矢印キー | ペイン間の移動（入力も可能） |
| `Ctrl+B` → `D` | ダッシュボードから抜ける |

## 仕組みの解説

### アーキテクチャ

`ws` コマンドは2つの tmux サーバーを使い分けています。

1. **デフォルトサーバー**: `ws add` で作成される各プロジェクトのセッション
2. **`ws-dash` サーバー**: ダッシュボード専用。`-L ws-dash` オプションで別サーバーとして起動

ダッシュボードの各ペインは `tmux attach` でデフォルトサーバーのセッションに接続（ミラーリング）しています。そのため **IDE 側で実行したコマンドや出力がダッシュボードにもリアルタイムで表示** されます。

### セッション名の管理

セッション名には `ws-` というプレフィックスが自動付与されます。これにより `ws` が管理するセッションと、ユーザーが手動で作った tmux セッションを区別しています。

```bash
# 内部的なセッション名
ws add api    # → tmux セッション名は "ws-api"
ws add web    # → tmux セッション名は "ws-web"
```

また、セッション名に含まれる `.` や `:` は `_` に自動変換されます。これは tmux のセッション名として使用できない文字への対応です。

### ダッシュボードのレイアウト

ペイン数に応じて自動的にレイアウトが選択されます。

- **3ペイン以下**: `even-horizontal`（横並び）
- **4ペイン以上**: `tiled`（タイル表示）

各ペインの上部にはプロジェクト名がボーダーとして表示されるため、どのペインがどのプロジェクトかが一目で分かります。

### ダッシュボードの更新

`ws add` や `ws rm` を実行すると、ダッシュボードが開いている場合は自動的に再構築されます。再構築の流れは以下の通りです。

1. 新しいウィンドウを作成し、現在のワークスペース一覧からペインを構築
2. 古いウィンドウを削除
3. レイアウトを再適用

この方式により、ダッシュボードを閉じることなくシームレスにペインが更新されます。

## まとめ

`ws` コマンドを使えば、複数プロジェクトのターミナル作業を一画面で俯瞰できるようになります。特に **複数のマイクロサービスを同時に開発・デバッグする場合** や、**各プロジェクトのビルド状況をリアルタイムで確認したい場合** に便利です。

tmux の基本的な機能を組み合わせただけのシンプルなスクリプトですが、日常の開発体験を大きく改善してくれるツールです。
